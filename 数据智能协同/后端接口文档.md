## 一、架构概述

任务4模块提供「自然语言 → SQL → 查询结果 → 智能解释 → 审计留痕」的完整闭环。前端可以通过以下两种方式调用：

| 调用方式 | 适用场景 | 示例 |
|---------|---------|------|
| **Python 直接导入**（推荐） | Django 视图/服务层 | `from airfoil_collab.collab import run_once` |
| **CLI 子进程调用** | 独立脚本、定时任务 | `python -m airfoil_collab run --question "..."` |

所有核心函数在 [`airfoil_collab/`](Airfoil-Engineering-Database-System/airfoil_collab/) 包中，均可直接 import 使用。

## 二、核心接口（Python API）

### 2.1 单次 NL2SQL 查询 + 审计 + 解释

```python
from airfoil_collab.collab import run_once, RunOutcome
from airfoil_collab.config import load_app_config

cfg = load_app_config()
out: RunOutcome = run_once(
    cfg,
    question="查询翼型编号为 NACA_2412 的当前有效版本号与创建时间。",
    schema_mode="strong",     # "strong" | "weak"
    username="web_user",      # 任意用户名，自动创建 user_account 记录
    do_explain=True,          # 是否调用 LLM 生成结果解释
    statement_timeout_ms=3000,
)
```

#### 返回结构 `RunOutcome`

| 字段 | 类型 | 说明 |
|------|------|------|
| `query_id` | `str \| None` | 审计链唯一 ID（UUID），可关联 `query_log` 表 |
| `audit.audit_status` | `str` | `"approved"` / `"needs_fix"` / `"rejected"` |
| `audit.error_types` | `list[str]` | 错误类型枚举（见 §5） |
| `audit.notes` | `str` | 审计备注 |
| `executed_sql` | `str \| None` | 最终执行的 SQL（已通过审计+添加 LIMIT） |
| `result_csv` | `str \| None` | 查询结果（CSV 格式，含表头） |
| `explanation_text` | `str \| None` | LLM 生成的结果解释（仅 `do_explain=True` 时有值） |
| `generated.obj` | `dict \| None` | LLM 原始返回 JSON（含 `sql`, `assumptions`, `risk_flags`） |

#### 前端调用示例（Django 视图）

```python
from django.http import JsonResponse
from airfoil_collab.collab import run_once
from airfoil_collab.config import load_app_config

def nl2sql_api(request):
    question = request.GET.get("q", "")
    if not question:
        return JsonResponse({"error": "missing q"}, status=400)

    cfg = load_app_config()
    out = run_once(cfg, question=question, do_explain=True)

    return JsonResponse({
        "query_id": out.query_id,
        "status": out.audit.audit_status,
        "sql": out.executed_sql,
        "data": out.result_csv,              # 前端自行解析 CSV
        "explanation": out.explanation_text,
        "error_types": out.audit.error_types,
    })
```

## 三、CLI 接口（供脚本/调试使用）

所有命令在 `Airfoil-Engineering-Database-System/` 目录下执行。

### 3.1 单次查询

```bash
python -m airfoil_collab run \
  --question "在 Re=3000000、攻角 2 条件下，列出升阻比最高的前 10 个翼型。" \
  --schema-mode strong \
  --username web_user \
  --print-result \
  --print-explain
```

**输出**：标准 JSON + 可选 CSV 结果 + 可选解释文本

## 四、数据库中的可用视图/函数（前端直接 SQL 查询时使用）

> 这些对象由任务1+2创建，前端 Django 可直接通过 ORM 或 raw SQL 调用。

### 4.1 当前有效版本视图

```sql
-- api.v_current_airfoil_version
-- 返回所有翼型的当前有效版本（is_current=true, status='valid', is_deleted=false）
SELECT airfoil_code, version_no, created_at, version_id
FROM api.v_current_airfoil_version
WHERE airfoil_code = 'NACA_2412';
```

### 4.2 封装 API 函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `api.get_airfoil_geometry(code, only_current, version_no)` | 获取翼型几何坐标点 | `SELECT * FROM api.get_airfoil_geometry('NACA_2412', true, NULL)` |
| `api.find_airfoils_by_condition(alpha, re, min_cl, max_cd, min_ld, only_current)` | 按工况筛选翼型 | `SELECT * FROM api.find_airfoils_by_condition(2, 3000000, NULL, NULL, NULL, true)` |
| `api.compare_airfoils_at_reynolds(codes[], re, only_current)` | 多翼型同 Re 对比 | `SELECT * FROM api.compare_airfoils_at_reynolds(ARRAY['NACA_2412','NACA_4421'], 3000000, true)` |
| `api.get_airfoil_performance_across_versions(code, alpha, re)` | 跨版本性能 | `SELECT * FROM api.get_airfoil_performance_across_versions('NACA_2412', 2, 3000000)` |
| `api.compare_airfoil_versions(code, alpha, re, ver_a, ver_b)` | 两版本对比 | `SELECT * FROM api.compare_airfoil_versions('NACA_2412', 2, 3000000, 1, 2)` |
| `api.list_airfoils_with_anomalies(only_current)` | 异常翼型列表 | `SELECT * FROM api.list_airfoils_with_anomalies(true)` |

### 4.3 主数据表（只读查询）

| 表名 | 关键字段 |
|------|---------|
| `airfoil` | `airfoil_id`, `airfoil_code`, `name`, `family` |
| `airfoil_version` | `version_id`, `airfoil_id`, `version_no`, `is_current`, `status` |
| `coordinate_point` | `version_id`, `point_order`, `x`, `y`, `surface` |
| `performance_record` | `record_id`, `version_id`, `condition_id`, `cl`, `cd`, `l_over_d` |
| `experiment_condition` | `condition_id`, `alpha_deg`, `reynolds_number` |
| `anomaly_record` | `record_id`, `version_id`, `anomaly_type` |

> ⚠️ **版本语义强制要求**：凡涉及「当前有效版本」的查询，必须通过 `api.v_current_airfoil_version` 选取 `version_id`，否则会被审计拦截。

## 五、审计状态与错误类型枚举

### 5.1 `audit_status`

| 值 | 含义 | 前端处理建议 |
|----|------|------------|
| `approved` | SQL 通过审计，已执行 | 正常展示结果 |
| `needs_fix` | 存在可修复问题（如缺少版本过滤） | 展示错误提示，建议用户细化问题 |
| `rejected` | 不可执行（危险语句/敏感数据/格式错误） | 展示拒绝原因，不展示任何数据 |

### 5.2 `error_types`（可能同时出现多个）

| 值 | 含义 |
|----|------|
| `unsafe_sql` | 包含 INSERT/UPDATE/DELETE/DROP 等危险语句 |
| `sensitive_data_request` | 尝试访问 `user_account` 等敏感表 |
| `version_semantics_error` | 访问 `airfoil_version` 但未锚定当前有效版本 |
| `aggregate_misuse` | GROUP BY 与 SELECT 列不一致 |
| `missing_join_condition` | 多表查询缺少连接条件 |
| `empty_sql` | LLM 未生成 SQL |
| `llm_call_failed` | DeepSeek API 调用失败 |

### 5.3 结果解释审计 `judgement`

| 值 | 含义 |
|----|------|
| `correct` | 解释基于结果，符合工程常识 |
| `incorrect` | 存在虚构/工程常识违规 |
| `unsupported` | 解释未充分基于结果 |

## 六、审计留痕表（前端可查询历史记录）

| 表名 | 说明 |
|------|------|
| `query_log` | 每次查询的执行记录（`query_id`, `user_id`, `sql_text`, `is_success`） |
| `nl2sql_audit` | NL→SQL 审计记录（`nl_question`, `generated_sql`, `audit_status`, `error_types_json`） |
| `result_explain_audit` | 结果→解释审计记录（`llm_explanation`, `judgement`, `issues_json`） |

三张表通过 `query_id` 关联，形成完整证据链。

## 七、⚠️ 本地化内容标注

以下内容因开发环境而异，**不应提交到团队 Git 仓库**（`.env` 已在 `.gitignore` 中）：

| 项目 | 文件 | 说明 |
|------|------|------|
| 🔴 `PSQL_PATH` | [`.env`](Airfoil-Engineering-Database-System/.env) | PostgreSQL `psql.exe` 的绝对路径（如 `D:\postgresql\bin\psql.exe`），每个开发者不同。**如果系统 PATH 已包含 psql，可删除此行** |
| 🔴 `PGPASSWORD` | [`.env`](Airfoil-Engineering-Database-System/.env) | 数据库密码，每个开发者本机可能不同 |
| 🔴 `DEEPSEEK_API_KEY` | [`.env`](Airfoil-Engineering-Database-System/.env) | DeepSeek API 密钥，每人独立申请 |
| 🟡 `pg_service.conf` | [`数据库设计/sql/pg_service.conf`](Airfoil-Engineering-Database-System/数据库设计/sql/pg_service.conf) | 包含 `host=localhost`，远程部署时需改为实际 IP |
| 🟡 数据库名 `airfoil_db` | `pg_service.conf` | 如果团队统一使用此库名则无需修改 |
| 🟢 其他代码文件 | `airfoil_collab/*.py` | 均为团队共享代码，无本地化依赖 |

### 新成员环境搭建步骤

```bash
# 1. 安装 PostgreSQL 15，确保 psql 在 PATH 中（或配置 PSQL_PATH）
# 2. 创建数据库
psql -U postgres -c "CREATE DATABASE airfoil_db;"

# 3. 复制 .env 模板，填入自己的密码和 API Key
#    （PSQL_PATH 仅在 psql 不在 PATH 时需要）

# 4. 初始化表结构 + 导入数据
python -m airfoil_collab bootstrap
python 数据库设计/sql/04_import_project_data.sql  # 或其他数据导入方式

# 5. 验证
python -m airfoil_collab run --question "查询翼型编号为 NACA_2412 的信息。" --no-explain
```
