# 数据库功能技术方案

## 1. 目标与范围

本方案依据《数据库功能需求文档.md》定义数据库侧的概念模型、逻辑模型、关键约束/索引、版本管理策略、异常数据识别落库方式与查询支撑能力，确保系统至少覆盖：

- 翼型基础信息管理
- 几何坐标管理（按翼型读取完整轮廓，区分上/下表面等标记）
- 性能数据管理（攻角、雷诺数等条件下的多条件查询）
- 数据版本管理（修订后保留版本而非覆盖）
- 异常数据识别（至少一类规则检测；本方案提供三类）
- 查询与分析（实现文档列出的 5 类查询）
- 数据导出与结果展示（数据库侧提供可导出的视图/查询与必要字段）

## 2. 技术选型与总体架构

- 数据库：PostgreSQL16
- 设计风格：面向版本的主数据模型（翼型主表 + 翼型版本表），几何/性能均引用 version\_id，避免覆盖历史数据
- 数据访问：通过视图/函数提供常用查询；复杂分析可由应用侧调用参数化 SQL
- 导出：通过视图 + COPY（或应用层流式导出）生成 CSV；图像/曲线由应用层渲染（数据库输出标准化序列数据）

## 3. 数据模型设计

### 3.1 概念实体与关系（文字 ER）

- AIRFOIL（翼型）：记录翼型稳定属性（编号、名称、类别、来源、生成方式、备注）
- AIRFOIL\_VERSION（翼型版本）：记录某翼型在某次修订下的数据版本与元信息
- AIRFOIL\_COORDINATE（几何坐标点）：记录某版本翼型的坐标点序列，并标记上/下表面（或其他）
- AIRFOIL\_PERFORMANCE（性能数据点）：记录某版本在不同工况（雷诺数、攻角等）下的气动性能
- ANOMALY\_RULE（异常规则）：可配置的规则定义（规则类型、参数、启用状态）
- ANOMALY\_RECORD（异常记录）：规则检测的结果落表，支持追溯与“识别含异常记录的翼型”

关系：

- AIRFOIL 1 — N AIRFOIL\_VERSION
- AIRFOIL\_VERSION 1 — N AIRFOIL\_COORDINATE
- AIRFOIL\_VERSION 1 — N AIRFOIL\_PERFORMANCE
- ANOMALY\_RULE 1 — N ANOMALY\_RECORD
- AIRFOIL\_VERSION 1 — N ANOMALY\_RECORD（异常可落在性能点或版本级）

### 3.2 命名与字段约定

- 主键：bigint（identity）或 uuid（二选一；本方案示例用 bigint identity）
- 时间：timestamptz
- 浮点：double precision；若需要高精度可用 numeric(18,10)
- 枚举：用 text + CHECK 或 PostgreSQL enum；为便于迁移，本方案用 text + CHECK
- 软删除：若需要可增加 is\_deleted；本方案默认不软删版本数据，避免破坏追溯

## 4. 逻辑表结构（建议 DDL）

> 说明：以下 DDL 为建议结构，字段可按实际业务扩展（如 Mach、温度、湍流度等工况参数）。

### 4.1 翼型基础信息（AIRFOIL）

```sql
create table if not exists airfoil (
  airfoil_id        bigint generated always as identity primary key,
  airfoil_code      text not null unique,
  airfoil_name      text not null,
  category          text,
  source            text,
  generation_method text,
  remark            text,
  created_at        timestamptz not null default now(),
  updated_at        timestamptz not null default now()
);
```

关键点：

- airfoil\_code 作为业务编号，唯一约束便于“给定翼型编号查询”
- updated\_at 可由触发器或应用维护（触发器属于实现细节，可选）

### 4.2 数据版本管理（AIRFOIL\_VERSION）

采用显式版本表的原因：

- 版本元信息（版本号、变更说明、父版本、校验信息、发布状态）天然与翼型主属性分离
- 几何与性能均可对齐到同一个 version\_id，支持“查询某翼型在不同版本下的性能差异”
- 能支持“同一翼型存在多个并行版本”（例如不同来源/不同生成方式的分支）以及回滚/对比

```sql
create table if not exists airfoil_version (
  version_id        bigint generated always as identity primary key,
  airfoil_id        bigint not null references airfoil(airfoil_id),
  version_no        integer not null,
  parent_version_id bigint references airfoil_version(version_id),
  change_note       text,
  data_source       text,
  status            text not null default 'draft' check (status in ('draft','released','archived')),
  created_by        text,
  created_at        timestamptz not null default now(),
  unique (airfoil_id, version_no)
);

create index if not exists idx_airfoil_version_airfoil on airfoil_version(airfoil_id, version_no desc);
```

版本号策略（推荐）：

- version\_no 从 1 递增
- 新版本创建时复制父版本的坐标/性能或引用外部导入批次（由业务流程决定）

### 4.3 几何坐标管理（AIRFOIL\_COORDINATE）

目标：能“按翼型读取完整轮廓”，并“区分上表面、下表面或其他必要标记”。

```sql
create table if not exists airfoil_coordinate (
  coord_id     bigint generated always as identity primary key,
  version_id   bigint not null references airfoil_version(version_id) on delete cascade,
  surface      text not null check (surface in ('upper','lower','other')),
  seq          integer not null,
  x            double precision not null,
  y            double precision not null,
  z            double precision,
  tag          text,
  unique (version_id, surface, seq)
);

create index if not exists idx_coord_version_surface_seq on airfoil_coordinate(version_id, surface, seq);
```

说明：

- seq 保证同一 surface 下点序一致，读出即可重建曲线
- z 为可选（2D 翼型为空；3D/展向剖面可扩展）
- tag 预留“其他必要标记”

### 4.4 性能数据管理（AIRFOIL\_PERFORMANCE）

目标：记录不同攻角、雷诺数等条件下的性能数据，并支持多条件查询（阈值筛选、曲线对比）。

```sql
create table if not exists airfoil_performance (
  perf_id     bigint generated always as identity primary key,
  version_id  bigint not null references airfoil_version(version_id) on delete cascade,
  reynolds    double precision not null,
  aoa_deg     double precision not null,
  mach        double precision,
  cl          double precision,
  cd          double precision,
  cm          double precision,
  ldr         double precision,
  source_run  text,
  created_at  timestamptz not null default now(),
  unique (version_id, reynolds, aoa_deg, mach)
);

create index if not exists idx_perf_version on airfoil_performance(version_id);
create index if not exists idx_perf_condition on airfoil_performance(reynolds, mach, aoa_deg);
create index if not exists idx_perf_threshold on airfoil_performance(reynolds, mach, ldr);
```

说明：

- ldr（升阻比）可直接存储或由 cl/cd 计算；为便于阈值查询与导出，建议落库并允许为空
- 不对 cd 做非负约束，以支持异常识别（阻力系数为负属于检测对象）

### 4.5 异常规则与异常记录

目标：实现至少一类规则检测；本方案提供三类规则并支持落表追溯。

```sql
create table if not exists anomaly_rule (
  rule_id      bigint generated always as identity primary key,
  rule_code    text not null unique,
  rule_name    text not null,
  rule_type    text not null check (rule_type in ('cd_negative','ldr_outlier','aoa_jump')),
  params       jsonb not null default '{}'::jsonb,
  enabled      boolean not null default true,
  created_at   timestamptz not null default now()
);

create table if not exists anomaly_record (
  anomaly_id   bigint generated always as identity primary key,
  rule_id      bigint not null references anomaly_rule(rule_id),
  version_id   bigint not null references airfoil_version(version_id) on delete cascade,
  perf_id      bigint references airfoil_performance(perf_id) on delete set null,
  severity     text not null default 'warning' check (severity in ('info','warning','critical')),
  message      text not null,
  details      jsonb not null default '{}'::jsonb,
  detected_at  timestamptz not null default now(),
  unique (rule_id, perf_id)
);

create index if not exists idx_anom_version on anomaly_record(version_id, detected_at desc);
```

## 5. 规则检测方案（SQL 逻辑）

### 5.1 规则 A：阻力系数为负（cd\_negative）

检测逻辑：

- 对同一 version\_id 的性能点，若 cd < 0 则插入异常记录

示例 SQL（增量写入，避免重复）：

```sql
insert into anomaly_record (rule_id, version_id, perf_id, severity, message, details)
select
  r.rule_id,
  p.version_id,
  p.perf_id,
  'critical',
  'cd 为负值',
  jsonb_build_object('cd', p.cd, 'reynolds', p.reynolds, 'aoa_deg', p.aoa_deg, 'mach', p.mach)
from airfoil_performance p
join anomaly_rule r on r.rule_type = 'cd_negative' and r.enabled
where p.cd is not null and p.cd < 0
on conflict (rule_id, perf_id) do nothing;
```

### 5.2 规则 B：升阻比异常偏离（ldr\_outlier）

检测思路（可配置）：

- 以同一版本 + 同一工况（reynolds、mach）为分组
- 对 ldr 做分位数/IQR 或均值±kσ 检测
- 参数通过 anomaly\_rule.params 配置（例如 method、k、min\_points）

实现方式：

- 数据库侧可用窗口函数计算均值与标准差（适合快速落地）
- 若需要稳健统计（分位数/IQR），使用 percentile\_cont

示例（均值±kσ，k 默认 3）：

```sql
with cfg as (
  select rule_id,
         coalesce((params->>'k')::double precision, 3.0) as k,
         coalesce((params->>'min_points')::int, 8) as min_points
  from anomaly_rule
  where rule_type = 'ldr_outlier' and enabled
),
stat as (
  select
    p.version_id, p.reynolds, p.mach,
    avg(p.ldr) as mu,
    stddev_samp(p.ldr) as sigma,
    count(*) as n
  from airfoil_performance p
  where p.ldr is not null
  group by p.version_id, p.reynolds, p.mach
)
insert into anomaly_record (rule_id, version_id, perf_id, severity, message, details)
select
  cfg.rule_id,
  p.version_id,
  p.perf_id,
  'warning',
  '升阻比偏离统计范围',
  jsonb_build_object(
    'ldr', p.ldr, 'mu', s.mu, 'sigma', s.sigma, 'k', cfg.k,
    'reynolds', p.reynolds, 'aoa_deg', p.aoa_deg, 'mach', p.mach
  )
from cfg
join airfoil_performance p on true
join stat s
  on s.version_id = p.version_id
 and s.reynolds = p.reynolds
 and s.mach is not distinct from p.mach
where p.ldr is not null
  and s.n >= cfg.min_points
  and s.sigma is not null
  and abs(p.ldr - s.mu) > cfg.k * s.sigma
on conflict (rule_id, perf_id) do nothing;
```

### 5.3 规则 C：邻近攻角性能变化不合理（aoa\_jump）

检测思路（可配置）：

- 同一版本 + 同一工况（reynolds、mach）按 aoa 升序
- 计算相邻点 Δaoa、Δcl、Δcd 或 Δldr
- 若 Δaoa 在“邻近范围”内（如 0.5°\~2°）且变化超过阈值则判为异常

示例（对 cl 变化率阈值）：

```sql
with cfg as (
  select rule_id,
         coalesce((params->>'max_dcl_per_deg')::double precision, 0.8) as max_dcl_per_deg,
         coalesce((params->>'max_daoa_deg')::double precision, 2.0) as max_daoa
  from anomaly_rule
  where rule_type = 'aoa_jump' and enabled
),
ordered as (
  select
    p.*,
    lag(aoa_deg) over w as prev_aoa,
    lag(cl)      over w as prev_cl
  from airfoil_performance p
  window w as (partition by version_id, reynolds, mach order by aoa_deg)
)
insert into anomaly_record (rule_id, version_id, perf_id, severity, message, details)
select
  cfg.rule_id,
  o.version_id,
  o.perf_id,
  'warning',
  '邻近攻角 cl 变化不合理',
  jsonb_build_object(
    'prev_aoa', o.prev_aoa, 'aoa_deg', o.aoa_deg,
    'prev_cl', o.prev_cl, 'cl', o.cl,
    'dcl_per_deg', (o.cl - o.prev_cl) / nullif(o.aoa_deg - o.prev_aoa, 0),
    'reynolds', o.reynolds, 'mach', o.mach
  )
from cfg
join ordered o on true
where o.prev_aoa is not null
  and o.cl is not null and o.prev_cl is not null
  and (o.aoa_deg - o.prev_aoa) > 0
  and (o.aoa_deg - o.prev_aoa) <= cfg.max_daoa
  and abs((o.cl - o.prev_cl) / nullif(o.aoa_deg - o.prev_aoa, 0)) > cfg.max_dcl_per_deg
on conflict (rule_id, perf_id) do nothing;
```

## 6. 查询与分析能力（SQL 示例）

### 6.1 给定翼型编号，查询其全部几何坐标

需求：给定 airfoil\_code，返回最新版本（或指定版本）完整轮廓点。

```sql
select
  a.airfoil_code,
  v.version_no,
  c.surface, c.seq, c.x, c.y, c.z, c.tag
from airfoil a
join airfoil_version v on v.airfoil_id = a.airfoil_id
join airfoil_coordinate c on c.version_id = v.version_id
where a.airfoil_code = $1
  and v.version_no = (
    select max(v2.version_no) from airfoil_version v2 where v2.airfoil_id = a.airfoil_id
  )
order by c.surface, c.seq;
```

### 6.2 给定工况条件，查询满足性能阈值的翼型

示例：给定 reynolds、mach、aoa\_deg 与阈值（cl>=、cd<=、ldr>=）。

```sql
select
  a.airfoil_code,
  v.version_no,
  p.reynolds, p.mach, p.aoa_deg, p.cl, p.cd, p.ldr
from airfoil_performance p
join airfoil_version v on v.version_id = p.version_id
join airfoil a on a.airfoil_id = v.airfoil_id
where p.reynolds = $1
  and p.mach is not distinct from $2
  and p.aoa_deg = $3
  and (p.cl is null or p.cl >= $4)
  and (p.cd is null or p.cd <= $5)
  and (p.ldr is null or p.ldr >= $6)
order by p.ldr desc nulls last;
```

### 6.3 比较多个翼型在同一雷诺数下的性能曲线

示例：输入翼型编号列表 + reynolds（+ mach），输出按翼型分组的曲线点。

```sql
select
  a.airfoil_code,
  v.version_no,
  p.aoa_deg, p.cl, p.cd, p.ldr
from airfoil a
join airfoil_version v on v.airfoil_id = a.airfoil_id
join airfoil_performance p on p.version_id = v.version_id
where a.airfoil_code = any($1)
  and p.reynolds = $2
  and p.mach is not distinct from $3
  and v.version_no = (
    select max(v2.version_no) from airfoil_version v2 where v2.airfoil_id = a.airfoil_id
  )
order by a.airfoil_code, p.aoa_deg;
```

### 6.4 查询某翼型在不同版本下的性能差异

示例：对同一工况（reynolds、mach、aoa\_deg）比较不同版本的 cl/cd/ldr。

```sql
select
  a.airfoil_code,
  v.version_no,
  p.reynolds, p.mach, p.aoa_deg,
  p.cl, p.cd, p.ldr
from airfoil a
join airfoil_version v on v.airfoil_id = a.airfoil_id
join airfoil_performance p on p.version_id = v.version_id
where a.airfoil_code = $1
  and p.reynolds = $2
  and p.mach is not distinct from $3
  and p.aoa_deg = $4
order by v.version_no;
```

若需要“差值”输出，可在应用层计算，或用窗口函数对相邻版本做差（略）。

### 6.5 识别含异常记录的翼型

```sql
select distinct
  a.airfoil_code,
  v.version_no,
  max(r.detected_at) over (partition by v.version_id) as last_detected_at
from anomaly_record r
join airfoil_version v on v.version_id = r.version_id
join airfoil a on a.airfoil_id = v.airfoil_id
order by a.airfoil_code, v.version_no;
```

## 7. 视图与函数（便于复用）

建议提供：

- v\_airfoil\_latest\_version：每个翼型最新版本映射（airfoil\_id -> version\_id）
- v\_airfoil\_coord\_latest：最新版本坐标点
- v\_airfoil\_perf\_latest：最新版本性能点
- fn\_detect\_anomaly(version\_id)：对指定版本执行规则检测（也可按批次执行）

示例视图（最新版本）：

```sql
create or replace view v_airfoil_latest_version as
select v.*
from airfoil_version v
join (
  select airfoil_id, max(version_no) as max_version_no
  from airfoil_version
  group by airfoil_id
) x on x.airfoil_id = v.airfoil_id and x.max_version_no = v.version_no;
```

## 8. 数据导出与结果展示支撑

### 8.1 CSV 导出

数据库侧建议通过视图/查询准备“扁平化结果”，由应用层调用 COPY：

- 导出几何点：airfoil\_code、version\_no、surface、seq、x、y、z、tag
- 导出性能点：airfoil\_code、version\_no、reynolds、mach、aoa\_deg、cl、cd、cm、ldr
- 导出异常：airfoil\_code、version\_no、rule\_code、severity、message、detected\_at、details

### 8.2 曲线展示

- 数据库输出曲线序列（按 aoa 排序的 cl/cd/ldr）
- 图像生成不在数据库内完成，由应用侧绘制折线图并可导出 PNG/SVG

## 9. 数据质量、性能与运维建议

- 批量导入：性能点/坐标点建议使用 COPY 或分批插入，并在导入完成后再执行异常检测与索引维护
- 索引策略：以“工况过滤”和“版本过滤”为核心建立组合索引；避免过度索引导致导入变慢
- 分区（可选）：当性能数据规模巨大时，可按 reynolds 或 version\_id 范围做分区表
- 备份：定期全量备份 + WAL 归档；关键版本发布前可做逻辑备份（pg\_dump）
- 权限：读写分离角色；异常规则配置与版本发布操作应受限

## 10. 验收对照（需求到设计）

- 翼型基础信息管理：airfoil 表覆盖名称/编号/类别/来源/生成方式/备注
- 几何坐标管理：airfoil\_coordinate 按 version\_id + surface + seq 保存点序并可整轮廓读取
- 性能数据管理：airfoil\_performance 支持 re/aoa/mach 多条件与阈值查询
- 数据版本管理：airfoil\_version 显式版本表；坐标与性能均引用 version\_id，保留历史
- 异常数据识别：提供 cd\_negative、ldr\_outlier、aoa\_jump 三类规则 + anomaly\_record 落表
- 查询与分析：提供 5 类查询 SQL 示例与建议视图
- 数据导出与展示：提供 CSV 导出字段与曲线序列输出方式
